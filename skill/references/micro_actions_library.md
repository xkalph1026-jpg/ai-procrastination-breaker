# 微行动模板库 (v1.3)

按7种拖延类型组织的微行动模板，每条 ≤5-10分钟。

---

## 1. 完美主义型 — 「先造垃圾，再装修」

| # | 微行动 | 时长 |
|---|--------|------|
| P1 | 打开文档，只写标题和姓名，关掉 | 2min |
| P2 | 写一个"烂版本"的第一句——故意写烂 | 3min |
| P3 | 画一个"草图"——只给自己看 | 5min |
| P4 | 用口语录一段语音描述要做什么 | 3min |
| P5 | 找一个已发布的同类作品，找出3个缺点 | 5min |
| P6 | 给自己发消息："我允许第一版很烂" | 1min |

> 海明威说第一稿都是垃圾。初稿的「初」字本来就可以很烂。

---

## 2. 任务模糊型 — 「先看，不分析」

| # | 微行动 | 时长 |
|---|--------|------|
| F1 | 打开原始数据/文档，只看前20行，列出3个观察 | 5min |
| F2 | 把关于任务的所有问题写下来（不回答，只列） | 5min |
| F3 | 找一篇同类型范文，只看目录结构，抄下来 | 5min |
| F4 | 给了解这个领域的朋友发消息问"该先搞清什么" | 2min |
| F5 | 用手机录音，用口语解释任务（像吐槽一样） | 3min |
| F6 | 画mindmap第一层（3-5个分支），不展开 | 5min |

> 模糊的反面不是清晰，是「先看一眼」。

---

## 3. 任务过大型 — 「切一块，只切一块」

| # | 微行动 | 时长 |
|---|--------|------|
| B1 | 大任务写纸上方，下画3框，每框填1个最小子任务 | 5min |
| B2 | 选最怕的子任务，只做第一个动作 | 5min |
| B3 | 选最有把握的子任务，做完它 | 5min |
| B4 | 翻目录，圈出最陌生的3个单元 | 3min |
| B5 | 子任务写便利贴上，选1张贴面前，其余放抽屉 | 3min |
| B6 | 告诉自己"今天只学1个知识点"，学完就停 | 5min |

> 吃掉大象的方法：一次一口。今天只吃一口。

---

## 4. 缺乏动力型 — 「找到最小的意义」

| # | 微行动 | 时长 |
|---|--------|------|
| M1 | 写下完成后的3个具体好处（不要"有成就感"） | 3min |
| M2 | 找出最容易做的10%，先做掉 | 5min |
| M3 | 设定奖励（"做完喝奶茶"），写下来贴屏幕上 | 2min |
| M4 | 任务与长期目标连线："我做XX，是为了能XX" | 3min |
| M5 | 计时5分钟，看能做多少——当游戏玩 | 5min |
| M6 | 想象如果不做，3天后什么局面 | 2min |

> 你不需要喜欢这件事，你只需要穿过它，到达想去的地方。

---

## 5. 决策疲劳型 — 「最小后悔原则」

| # | 微行动 | 时长 |
|---|--------|------|
| D1 | "不选哪个会最后悔？" → 选它 | 3min |
| D2 | 设3个筛选标准，用标准筛 | 5min |
| D3 | 选后写"一句话描述"发给相关人确认 | 5min |
| D4 | 扔硬币——空中时希望哪面朝上就是答案 | 1min |
| D5 | 其他选项存"暂存档案"——不是放弃，是存档 | 3min |
| D6 | "10天后我还会在乎这个选择吗？" | 2min |

> 「不选」本身已经是选择了——而且是最差的那个。

---

## 6. 中断重启型 — 「读旧代码，不写新代码」

| # | 微行动 | 时长 |
|---|--------|------|
| R1 | 找到之前的文件，从头到尾读一遍，只读不写 | 5min |
| R2 | 给之前写的每行/每段写一句中文注释 | 10min |
| R3 | 找一个小bug或小改进，只修那个 | 5min |
| R4 | 写下"还记得的部分"和"完全忘了的部分" | 5min |
| R5 | 重新跑一遍之前能跑的代码/流程 | 3min |
| R6 | 打开README或笔记，只读第一节 | 3min |

> 大脑缓存清空了，但硬盘还在。你只需要重新"加载"一下。

---

## 7. 多任务并发型 — 「先救最急的」

| # | 微行动 | 时长 |
|---|--------|------|
| T1 | 把所有任务写纸上 → 标紧迫度(1-3) → 只做最紧迫的最小一步 | 5min |
| T2 | 问自己"哪个不做的后果最严重？" → 只做那个 | 3min |
| T3 | 找有没有可以委托或推迟的 → 划掉 | 3min |
| T4 | 每个任务只做"下一步动作"（不深入）→ 给大脑卸货 | 5min |
| T5 | 设定"今天只攻1个"——其他的明天再说 | 2min |
| T6 | 紧急度×难度矩阵 → 先做紧急+简单的 | 5min |

> 不是你做不完，是你想同时做完。先救那个着火的。

## 通用模板

| # | 微行动 | 时长 |
|---|--------|------|
| G1 | 5分钟倒计时，做任何和任务有关的事，铃声一响就停 | 5min |
| G2 | 手机放另一个房间，只盯任务材料看1分钟 | 1min |
| G3 | 告诉一个人"我要开始做XX了" | 2min |
| G4 | 写下"我现在感觉____，因为____"（只写感觉） | 3min |

---

## 设计原则

1. **≤5分钟** — 超过就不是微行动
2. **有明确"做完"信号** — 不需要判断质量
3. **不依赖意志力** — 小到不需要下决心
4. **失败了也安全** — 太小了，没挫败感

**微行动的目的不是完成任务，是打破「不启动」的魔咒。启动之后，惯性会带你走很远。**
